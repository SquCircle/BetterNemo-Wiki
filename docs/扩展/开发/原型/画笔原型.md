# 画笔原型
**画笔原型** （Brush.prototype）是目前BN已扩展的原型中方法数量最多的原型。它用于控制舞台的**HTML Canvas** 。

官方对于Canvas的应用是非常不全面的，光是只看画笔积木盒都能看出来了——

- 设置画笔颜色，要么用Nemo内置取色器（ *//** **难用的钥匙** *），要么就是用颜色值/饱和度/亮度来挨个设置属性（也就是HSL模式）。燃鹅，查看代码可以发现，取色器的返回值居然是正宗的HEX字符串`"#ccffcc"`！

- 另一个例子是，设置填充颜色没有透明度设置，原因是在实现中，把Canvas Context的fillStyle属性硬编码为了`"#"`` + ``this``.stroke_color`，而JS只支持六位的HEX，没有表示Alpha的后两位...

- 当然还有文字印章，只有文本、文本尺寸和水平相对位置三个属性，字体、字重（硬编码为了bold）、纵向相对位置（同样固定为center）等自定义属性被完全舍去了。

从这三个例子可以看出，HTML Canvas简直是被原生Nemo浪费了，因此我们可以扩展的画笔内容就相当多了。

# 从调用开始 - 复活图像印章

接下来我们开始学习如何自定义一个画笔原型。首先我们需要了解一下Nemo作品的运行顺序。

1. 当你点下开始时，解释器的事件监听器开始等待事件触发。如果你拼接了“**当 开始 被点击时** ”，那此事件会立即触发，然后运行下面的积木。

2. 积木运行时，解释器逐行解释，获取积木的类型，然后去调用它对应的**积木函数** 。

3. 积木函数运行时，获取参数并处理后，传给它调用的**角色原型方法** 。

4. 原型方法再接收参数再运行，并返回值。

而我们要写的就是第二和第三步。在翻找代码时，我们无意间发现了官方写好了的**图像印章** 方法，而积木盒中我们并没有找到这样一块积木。所以我们可以来尝试将它“复活”。

## 积木定义

我们在blocks.js中的`penBlocks`常量里定义一个`bn_draw_image_stamp`积木：

```javascript
const penBlocks = [
  {
    // 积木名
    type: "bn_draw_image_stamp",
    // 描述定义
    message0: "[B] 图像印章",
    // 参数定义
    args0: [],
    // 积木颜色
    colour: "%{BKY_PEN_HUE}",
    // 封装好了的块积木通用属性
    ...method_block,
  }
]
```

然后来到domain-functions.js。添加`bn_draw_image_stamp`的积木函数：

```javascript
regDomainFunction("bn_draw_image_stamp", (params, uuid, uuid2, utils) => {
  /*
  画笔万能头。用于检测是否为Actor。
  画笔类积木最好都加上这两行来错误处理，因为获取Brush需要是Actor的子类。
  如果不是就会抛错。
  */
  var actor = get_stage_target(uuid2);
  if (!actor) return;
  
  // 调用图像印章的原型
  actor.get_brush().draw_image_stamp();
});
```

`actor.get_brush().draw_image_stamp();`这行代码中，`get_brush()`返回一个 Brush 实例，然后调用其原型方法`draw_image_stamp()`。Actor 是所有角色对象的基类，任何继承自 Actor 的对象都可以通过`get_brush()`方法获取其关联的画笔对象。`draw_image_stamp()`是 Brush 原型上定义的图像印章绘制方法。

最后我们去toolbox.js中把积木添加进积木盒。这里我们添加至画笔积木盒：

```javascript
// 删除第六积木盒（可以在角色的编辑界面数一数）的最后一项，也就是结束线
Blockly.mainWorkspace.toolbox_.children_[5].blocks.pop();
// 定义新积木
Blockly.mainWorkspace.toolbox_.children_[5].blocks = Blockly.mainWorkspace.toolbox_.children_[5].blocks.concat(
    [
      // 加入
      u.block("bn_draw_image_stamp")
    ].map((block) => str2xml(block)),
);
```

至此我们就成功复活了图像印章，这个Nemo迟迟没有加入的东西。

*/* 据说Nemo代码很大一部分来自Kitten，这个非技术原因的不加入就很何意味了... */*

# 自定义原型-文字印章

接下来我们将扩展文字印章积木。直到今天，大家都还在抱怨文字印章样式太少；与其等官方更新，不如我们自己把它整出来。

BN的原型定义写在prototype-inject.js里。我们也将在这里编写我们新的文字印章原型方法。

我们不推荐修改自带的原型方法，因为这会导致一些意想不到的问题。因此我们选择添加一个新的原型方法来打印文字。

添加原型使用`对象名.prototype.方法名 = ``function`` (参数...) {方法内容}`。

首先我们在prototype-inject.js中挑选一个风水宝地，另起一行，写入：

```javascript
Brush.prototype.better_draw_text_stamp = function (text, color, style, weight, size, font, align, base_line, rotation) {
  
}
```

其中，better_draw_text_stamp是原型方法名，text、color、style、weight、size、font、align、base_line、rotation是原型方法要接收的参数。至于为什么有这么多，当然是为了更多的自定义效果。

每个参数的作用：

| 参数名 | 作用/目的 |
| --- | --- |
| text<br> | 要打印的文本。例如：<br>`"Hello, world!"` |
| color | 文本颜色。支持HEX、CSS的`rgba()`和`hsla()`。 |
| style | 字体样式：正常、斜体、倾斜体。 |
| weight | 字重。支持字符串和数字。 |
| size | 文本大小。和原版一样。 |
| font | */* 最值得添加的属性！*/*<br>文本字体。得是Webview支持的字体。 |
| align | 水平对齐方式。 |
| base_line | 垂直对齐样式。 |
| rotation | 打印方向（等于角色方向） |

接下来在大括号里编写方法逻辑。

```javascript
// 一个画笔原型万能头：
// 获取上下文
var ctx = this.ctx;
// 如果没有上下文，返回
if (!ctx) {
  return;
}
// 获取舞台大小
var _a2 = this.app.get_app().view;
var width2 = _a2.width;
var height2 = _a2.height;
// 舞台中心点的Canvas坐标
var center_x = width2 / 2;
var center_y = height2 / 2; 
// 获取角色坐标
var _a = this.app.get_app().stage.toGlobal(this.actor.position);
var x = _a.x;
var y = _a.y;
```

接下来就可以用这个ctx在舞台上进行绘制操作了。

我们先设置一下ctx.font属性：

```javascript
ctx.save(); // 保存画笔状态
// 样式 字重 大小（单位px） 字体（如果字体缺失，则依次使用Arial和微软雅黑）
ctx.font = style + ' ' + weight + ' ' + size + "px " + font + ", Arial, 'Microsoft YaHei'";
```

然后是ctx.fillStyle。由于篇幅，我们此处只示范纯色。渐变可以大家自行研究。

```javascript
ctx.fillStyle = color;
// 使用角色的画笔透明度属性
ctx.globalAlpha = this.alpha;
```

设置ctx位置方向状态：

```javascript
// 对齐方式
ctx.textBaseline = base_line;
ctx.textAlign = align;
// 方向与位置
ctx.translate(x, y);
ctx.rotate(stamp_rotation);
```

然后可以进行打印了：

```javascript
ctx.fillText(text, 0, 0);
// 恢复上下文状态（即之前save()的）
ctx.restore();
// 更新Canvas
this.actor.parent_scene.should_update_brush();
```

至此我们的新的原型方法就完成了。之后在积木定义里调用它就可以让积木打印出文字了。

*/* 不要忘记参数，这会导致积木盒打不开或者其他报错。此外要明确参数的数据类型。 */*

## 参考代码

  // 一个画笔原型万能头：
  // 获取上下文
  var ctx = this.ctx;
  // 如果没有上下文，返回
  if (!ctx) {
    return;
  }
  // 获取舞台大小
  var _a2 = this.app.get_app().view;
  var width2 = _a2.width;
  var height2 = _a2.height;
  // 舞台中心点的Canvas坐标
  var center_x = width2 / 2;
  var center_y = height2 / 2;
  // 获取角色坐标
  var _a = this.app.get_app().stage.toGlobal(this.actor.position);
  var x = _a.x;
  var y = _a.y;
  
  ctx.save(); // 保存画笔状态
  // 样式 字重 大小（单位px） 字体（如果字体缺失，则依次使用Arial和微软雅黑）
  ctx.font = style + ' ' + weight + ' ' + size + "px " + font + ", Arial, 'Microsoft YaHei'";
  ctx.fillStyle = color;
  // 使用角色的画笔透明度属性
  ctx.globalAlpha = this.alpha;
  
  // 对齐方式
  ctx.textBaseline = base_line;
  ctx.textAlign = align;
  // 方向与位置
  ctx.translate(x, y);
  ctx.rotate(stamp_rotation);
  
  ctx.fillText(text, 0, 0);
  // 恢复上下文状态（即之前save()的）
  ctx.restore();
  // 更新Canvas

}
